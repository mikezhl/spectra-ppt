## Page 1: Company Purpose（封面 + 一句话定位）

**标题：** Spectra

**一句话定位：** AI 时代的学校操作系统——让每一条教育数据都能被轻松录入、被深度理解、被每个人即时使用。

**副标题：** 智能教务管理系统

设计要求：极简白色背景，大标题居中，底部有一个可交互的 "Start" 按钮。

## Page 2: Problem（四方痛点）

标题：四方痛点

展示四个角色各自面临的核心问题：

**老师：**
- 家长沟通低效，耗费大量精力
- 学情跟踪困难——成绩在教务系统、表现在脑子里、活动记录在纸上，无法形成完整学生画像
- 琐事繁多——写报告、填表、发通知、写评语花费大量时间

**家长：**
- 无法实时了解孩子的表现，学校反馈不及时
- 收到的反馈模糊、评价敷衍、数据不足
- 长期信息滞后

**管理层：**
- 数据分散，查询困难
- 想看全校学情趋势，要手动拉表、人工汇总
- 决策靠经验，缺乏数据支撑

**学生：**
- 高中生活中大量有价值的经历没有被系统性记录，毕业后难以回忆
- 作业千人一面，缺乏个性化学习路径

## Page 3: Why Now?（为什么是现在）— 【新增页】

标题：为什么是现在？
副标题：AI 已经改变了几乎所有行业，但教育是渗透最慢的领域之一

核心论点：AI 技术已成熟并广泛应用于各行业，但教育领域——尤其是学校教务管理——几乎未被 AI 触及。现在是填补这个空白的最佳时机。

三个驱动力：

**1. 技术成熟 + 成本骤降**
- 2024 年前：NLP 无法可靠理解非结构化教育信息（语音评价、自由文本反馈）
- 2025-2026：GPT-4 级别模型 + Agent 框架让自然语言理解、多维数据关联、主动推理成为标配能力
- 推理成本每 6 个月下降 50%+，大规模部署已经经济可行
- 各行业已大规模采用 AI，但教育是 AI 渗透最少的领域之一——巨大的空白等待填补

**2. 国家政策强力推动**
- 2024《教育强国建设规划纲要（2024-2035）》：中央将 AI+教育融合列为国家战略
- 2025.04《教育部等九部门关于加快推进教育数字化的意见》：明确"智能化"方向
- 2026.04《"人工智能+教育"行动计划》（五部门联合）：直接鼓励 AI 赋能教育教学和学校管理
- 政策体系完整，从顶层设计到落地规范全部就位

**3. 市场窗口：国际学校的数字化空白**
- 1,000+ 所国际/高端民办学校，大多还在用 ManageBac（系统简陋）、校宝在线（偏体制内）等上一代工具
- 没有一个产品同时解决"数据录入门槛高"和"数据录进去了用不起来"两个问题
- 先发优势 = 数据壁垒。窗口期约 18 个月，进去了就出不来

## Page 4: 我们处在的位置（时代演进）

标题：教育信息化的三个时代

三列对比：

| 纸质时代（~2010） | 信息化时代（2010-2025） | Agent 时代（现在） |
|---|---|---|
| 成绩手写，评语手写 | 纸质→电子表格 | 输入：语音/自然语言，AI 自动理解归档 |
| 家长会一学期一次 | 微信沟通 | 输出：对话式查询，想问什么问什么 |
| 所有信息在纸上和脑子里 | 解决了存储，没解决理解 | AI 关联多维数据、主动发现问题 |
| | 做到了数字化，没做到智能化 | 每个学生有班主任 Agent，每个老师有秘书 Agent |

底部时间轴进度条，"Agent 时代"高亮。

## Page 5: Solution — Data Layer（数据层）

标题：构建更完善的数据库
副标题：传统系统只录入成绩和出勤，我们覆盖全维度信息

2×2 网格，四种数据类型：

1. **课堂表现**（语音录入）—— 老师语音快速录入，AI 自动识别学生、分类标签
2. **课后作业**（批改 + 个性化出题）—— 自动批改、错题本、知识点分析、根据掌握程度自动生成个性化练习题
3. **考试成绩**（智能识别）—— 智能识别分数与错题，个人报告 + 班级对比 + 知识点热力分析
4. **碎片化信息**（轻量录入）—— 社团、体育、日常，30秒语音录入，自动关联学生档案

底部流程条：录入 → LLM 识别学生 → 归档至个人档案

## Page 6: Solution — Agent Layer（Agent 层）

标题：每个人都有专属 Agent
副标题：Agent 访问数据库，为家长、老师和学生提供个性化服务

三列：

**家长端 Agent（微信小程序）：**
- 随时查询成绩/作业/课堂表现
- 每周学情周报推送
- 成绩趋势可视化
- 严格数据隔离

**老师端 Agent（查询+分析+生成）：**
- 学生全维度画像
- 班级分析：知识点薄弱、学科表现
- 自动生成评语和报告
- 30秒语音录入
- 学情预警

**学生端 Agent（成长记录+个性化学习）：**
- 自主创建事件：社团、比赛、项目
- 所有经历永久保存，毕业后可回看
- 个性化作业
- 升学时一键导出活动档案

## Page 8: Competition（竞争格局）— 【新增页】

标题：竞争格局
副标题：没有产品同时解决"录不进"和"问不出"

**三个主要竞品分析：**

**校宝在线：**
- 主打体制内学校，近年尝试拓展体制外
- 偏教学辅助和行政管理，不具备 AI Agent 能力
- 国际化适配弱，不支持 IB/AP 等评估体系的多维数据需求
- 定位：传统教务 SaaS

**校校（XiaoXiao）：**
- 主打培训机构，和机构合作多，与国际学校合作少
- 做小班制课程管理 + 教务系统深度定制
- 与学校的绑定程度不如我们（Design Partner 模式）
- 定位：机构管理工具

**ManageBac：**
- 国际学校中使用较广，但系统非常简陋
- 仅满足基础的课程管理和成绩记录
- 无 AI 能力，无数据分析，无家校沟通功能
- 目前用 ManageBac 的学校普遍对其不满意
- 定位：上一代国际学校管理系统

**Spectra 的差异化定位：**

| 维度 | 校宝在线 | 校校 | ManageBac | Spectra |
|------|---------|------|-----------|---------|
| 目标客户 | 体制内学校 | 培训机构 | 国际学校 | 国际/高端学校 |
| AI 能力 | 无 | 无 | 无 | Agent 全链路 |
| 数据录入 | 表单填写 | 表单填写 | 表单填写 | 语音/自然语言 |
| 数据分析 | 固定报表 | 基础统计 | 几乎无 | AI 多维关联分析 |
| 个性化服务 | 无 | 无 | 无 | 每人专属 Agent |
| 学校绑定深度 | 中 | 低 | 中 | 极高（Design Partner） |

**核心差异化一句话：**
我们是唯一一个**从数据采集到 Agent 服务全链路打通**的产品。传统系统解决了存储，没解决理解；Spectra = 教育数据基础设施 + AI Agent 服务层。
